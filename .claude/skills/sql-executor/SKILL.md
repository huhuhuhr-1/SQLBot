# SQL Executor

## Overview

Execute SQL queries across multiple database types with comprehensive security validation, connection management, and JSON result formatting. This skill bridges the gap between SQL generation (sql-generator skill) and data visualization (echarts-renderer skill) by providing safe, reliable SQL execution.

## Core Capabilities

### 1. Multi-Database Support

Execute queries on 11+ database types:
- **Relational**: MySQL, PostgreSQL, SQL Server, Oracle
- **Analytical**: ClickHouse, Doris, StarRocks, Redshift
- **Chinese Databases**: DM (达梦), KingBase
- **Search Engine**: Elasticsearch

See [databases.md](references/databases.md) for complete configuration details.

### 2. Security Validation

Built-in SQL injection detection and read-only mode enforcement:
- Detects SQL injection patterns (semicolons, comments, boolean-based injection)
- Blocks write operations when read-only mode is enabled
- Validates query structure before execution
- Provides detailed error messages for blocked queries

### 3. Connection Management

Robust connection handling with configurable parameters:
- Query timeout control (default: 30 seconds)
- Maximum row limits to prevent excessive data transfer
- Automatic connection cleanup
- Support for connection pooling and schema selection

### 4. Result Formatting

Consistent JSON output format:
```json
{
  "success": true,
  "fields": ["column1", "column2", "column3"],
  "data": [
    {"column1": "value1", "column2": 123, "column3": 1.23},
    {"column1": "value4", "column2": 456, "column3": 4.56}
  ],
  "row_count": 2
}
```

## Quick Start

### Basic Query Execution

```python
from scripts.execute_sql import execute_sql

result = execute_sql(
    db_type="mysql",
    host="localhost",
    port=3306,
    username="root",
    password="password",
    database="test_db",
    sql="SELECT * FROM users LIMIT 10",
    timeout=30,
    read_only=True
)

if result["success"]:
    print(f"Retrieved {result['row_count']} rows")
    print(result["data"])
else:
    print(f"Error: {result['error']}")
```

### SQL Validation

```python
from scripts.sql_validator import SQLValidator

validator = SQLValidator(read_only=True)
validation = validator.validate("SELECT * FROM users")

if validation["valid"]:
    print("SQL is safe to execute")
else:
    print(f"Validation errors: {validation['errors']}")
```

```python

from scripts.execute_sql import execute_sql

# Step 1: Validate
validator = SQLValidator(read_only=True)

if not validation["valid"]:
    return {"error": "SQL validation failed", "details": validation["errors"]}

# Step 2: Execute
result = execute_sql(...)
```

## Security Features

### SQL Injection Detection

Automatically detects and blocks:
- Multiple statements (semicolons)
- SQL comments (`--`, `/* */`)
- Boolean-based injection (`OR 1=1`, `AND 1=1`)
- UNION-based injection
- Extended stored procedures
- Dynamic SQL execution

### Read-Only Mode Enforcement

When `read_only=True` (default):
- Only allows SELECT queries
- Blocks: DROP, DELETE, INSERT, UPDATE, ALTER, CREATE, TRUNCATE, etc.
- Requires query to start with: SELECT, WITH, SHOW, DESCRIBE, EXPLAIN

### Query Timeout Control

Prevents long-running queries:
- Default timeout: 30 seconds
- Configurable per query
- Automatic connection cleanup on timeout

### Result Size Limits

Prevents excessive data transfer:
- `max_rows` parameter limits returned rows
- Recommended defaults: 100-10000 rows depending on use case

## Configuration Parameters

### Required Parameters

- `db_type`: Database type identifier (mysql, pg, sqlServer, oracle, ck, doris, starrocks, redshift, dm, kingbase, es)
- `host`: Database hostname or IP
- `port`: Database port
- `username`: Database username
- `password`: Database password
- `database`: Database name
- `sql`: SQL query to execute

### Optional Parameters

- `timeout`: Query timeout in seconds (default: 30)
- `db_schema`: Schema name (PostgreSQL specific)
- `extra_jdbc`: Extra JDBC parameters
- `max_rows`: Maximum rows to return
- `read_only`: Enforce read-only mode (default: True)

## Error Handling

### Common Error Scenarios

**Connection failed:**
```json
{
  "success": false,
  "error": "Access denied for user 'root'@'localhost'"
}
```

**SQL validation failed:**
```json
{
  "success": false,
  "error": "SQL validation failed",
  "details": [
    "Multiple statements detected",
    "Write operation detected: DROP"
  ]
}
```

**Query timeout:**
```json
{
  "success": false,
  "error": "Query execution timeout",
  "type": "TimeoutError"
}
```

### Best Practices

1. **Always validate SQL** before execution in production environments
2. **Use read-only mode** unless write operations are explicitly required
3. **Set appropriate timeouts** based on query complexity
4. **Limit result size** with `max_rows` parameter
5. **Handle errors gracefully** with try-catch blocks
6. **Log validation warnings** for security auditing

## Performance Optimization

### Query Optimization Tips

1. **Use specific columns**: Avoid `SELECT *`
2. **Add LIMIT clauses**: Restrict result set size
3. **Create indexes**: Ensure WHERE and JOIN columns are indexed
4. **Optimize JOINs**: Use appropriate join types
5. **Use WHERE clauses**: Filter data early

### Timeout Recommendations

- Simple queries: 10-30 seconds
- Queries with JOINs: 30-60 seconds
- Analytical queries: 60-300 seconds

Use `SQLValidator.suggest_timeout()` to auto-suggest timeout based on query complexity.

## Troubleshooting

### Database Driver Not Installed

```bash
# Install required drivers
pip install pymysql psycopg2-binary pymssql oracledb clickhouse-connect redshift_connector dmPython
```

### Connection Timeout

- Verify host and port are correct
- Check firewall rules
- Ensure database is running
- Increase timeout parameter

### SQL Validation Failed

- Review validation errors
- Check for restricted keywords
- Ensure query starts with SELECT/WITH/SHOW/DESCRIBE/EXPLAIN
- Disable read-only mode only if write operations are required

- You need data for echarts-renderer visualizations

- You need to manage database connections safely
