### 数据源权限管理

#### 行级权限
```sql
-- 用户只能看到自己部门的数据
SELECT * FROM sales 
WHERE department_id = (
  SELECT department_id FROM users WHERE id = :user_id
);
```

#### 列级权限
```sql
-- 普通用户不能看到敏感字段
SELECT id, name, email, department_id 
FROM users 
WHERE id = :user_id;
```

## 🤖 智能问答

### 基本使用方法

#### 1. 自然语言查询

```
用户: "显示上个月的销售数据"
SQLBot: SELECT * FROM sales 
        WHERE date >= date_trunc('month', current_date - interval '1 month')
        AND date < date_trunc('month', current_date);
```

#### 2. 复杂业务查询

```
用户: "分析各个产品类别的销售趋势，按季度统计"
SQLBot: SELECT 
          category,
          date_trunc('quarter', sale_date) as quarter,
          SUM(amount) as total_sales,
          COUNT(*) as order_count
        FROM sales s
        JOIN products p ON s.product_id = p.id
        GROUP BY category, quarter
        ORDER BY category, quarter;
```

### 高级功能

#### 1. 上下文理解

```
用户: "这个数据比上个月怎么样？"
SQLBot: 基于之前的对话上下文，自动比较当前数据与上个月的数据
```

#### 2. 智能建议

```
用户: "我想分析用户行为"
SQLBot: 建议分析维度：
        - 用户活跃度
        - 功能使用频率
        - 用户留存率
        - 转化漏斗
```

#### 3. 数据解释

```
用户: "为什么这个月的销售额下降了？"
SQLBot: 分析可能原因：
        - 季节性因素
        - 营销活动变化
        - 产品问题
        - 竞争影响
```

### 最佳实践

#### 1. 提问技巧

- **具体明确**: "显示北京地区 Q1 的销售数据" 而不是 "显示销售数据"
- **分步查询**: 复杂问题分解为多个简单查询
- **提供上下文**: 说明业务背景和需求

#### 2. 结果验证

- 检查数据量是否合理
- 验证关键指标的计算
- 对比历史数据进行验证

## 📈 图表生成

### 支持的图表类型

- **基础图表**: 柱状图、折线图、饼图、散点图
- **高级图表**: 热力图、雷达图、漏斗图、桑基图
- **组合图表**: 多轴图、混合图表
- **地理图表**: 地图、地理散点图

### 自动图表生成

#### 1. 智能推荐

SQLBot 会根据数据特征自动推荐合适的图表类型：

- **时间序列数据** → 折线图
- **分类对比数据** → 柱状图
- **占比数据** → 饼图
- **相关性数据** → 散点图

#### 2. 图表优化

- **颜色搭配**: 自动选择美观的配色方案
- **标签布局**: 智能调整标签位置避免重叠
- **坐标轴**: 自动优化坐标轴范围和刻度

### 自定义图表

#### 2. 交互功能

- **数据钻取**: 点击图表元素查看详细数据

## 🎛️ 仪表板管理

### 创建仪表板

#### 1. 基本步骤

1. **命名仪表板**: 给仪表板起一个描述性的名称
2. **选择布局**: 选择合适的网格布局
3. **添加组件**: 拖拽图表和控件到画布
4. **配置样式**: 设置颜色、字体、间距等

#### 2. 组件类型

- **图表组件**: 各种类型的图表
- **文本组件**: 标题、说明文字
- **筛选器**: 下拉选择、日期选择、搜索框
- **统计卡片**: 关键指标展示

### 仪表板设计原则

#### 1. 信息层次

- **重要信息**: 放在左上角，使用大字体和醒目颜色
- **次要信息**: 放在右侧或下方，使用较小字体
- **辅助信息**: 放在底部，使用最小字体

#### 2. 视觉平衡

- **对称布局**: 左右对称的布局更加平衡
- **色彩协调**: 使用统一的配色方案
- **间距一致**: 保持组件间的统一间距

#### 3. 交互友好

- **操作简单**: 减少用户操作步骤
- **反馈及时**: 操作后立即显示结果
- **错误提示**: 清晰的错误信息和解决建议

### 分享和协作

#### 1. 权限设置

- **查看权限**: 谁可以查看仪表板
- **编辑权限**: 谁可以修改仪表板
- **分享权限**: 谁可以分享仪表板

#### 2. 导出功能

- **图片导出**: PNG、JPG 格式
- **PDF 导出**: 包含完整仪表板内容
- **数据导出**: CSV、Excel 格式

## 👥 用户管理

### 用户角色

#### 1. 系统管理员

- **权限**: 所有功能的管理权限
- **职责**: 系统配置、用户管理、数据源管理

#### 2. 数据分析师

- **权限**: 数据查询、图表创建、仪表板管理
- **职责**: 数据分析、报表制作、业务洞察

#### 3. 普通用户

- **权限**: 数据查看、基础查询
- **职责**: 日常数据查看、简单分析

### 权限配置

#### 1. 数据权限

```sql
-- 用户只能访问指定部门的数据
CREATE POLICY department_policy ON sales
FOR SELECT USING (
  department_id IN (
    SELECT department_id FROM user_departments WHERE user_id = current_user_id()
  )
);
```

#### 2. 功能权限

- **查询权限**: 可以执行哪些类型的查询
- **导出权限**: 是否可以导出数据
- **管理权限**: 是否可以管理数据源和用户

### 安全设置

#### 1. 密码策略

- **复杂度要求**: 至少 8 位，包含大小写字母、数字、特殊字符
- **过期策略**: 定期要求修改密码
- **锁定策略**: 多次失败登录后锁定账号

#### 2. 访问控制

- **IP 白名单**: 限制访问来源
- **时间限制**: 限制访问时间
- **会话管理**: 自动超时和强制登出
